# Security Policy

## Reporting a Vulnerability

We take the security of this project seriously. If you discover a security vulnerability, please report it responsibly.

**Do not** report security vulnerabilities through public GitHub issues. Instead, please send an email to **saksham1991999@gmail.com**.

We will acknowledge receipt of your report within 48 hours and work with you to understand and address the issue promptly.

## Scope

Since this repository contains educational course materials and example Django applications, security issues may include:

- Insecure code patterns in example applications
- Exposed credentials or secrets
- Dependency vulnerabilities in course projects

## Process

1. **Report**: Email details of the vulnerability to saksham1991999@gmail.com
2. **Acknowledgment**: We will confirm receipt within 48 hours
3. **Investigation**: We will investigate and determine the impact
4. **Resolution**: We will work on a fix and release it promptly
5. **Disclosure**: We will coordinate disclosure of the vulnerability

We appreciate your help in keeping this project and its users safe!
